Curso Git & GitHub

1. Criar uma conta no GitHub;

2. Instalar o Git no computador;

3. No terminal do próprio VS Code, executar os seguintes comandos:

abra seu projeto no VS, e execute em seguida:

'git init' - caso dê algum erro, garantuir que a variável de ambiente PATH esteja correta no seu computador.

'git add' - em seguida, esse servirá para adicionar todos os arquivos do projeto, para não precisar adicionar um por um, utilizaremos um atalho: 'git add .' o "." fará com que todos os arquivos do projeto sejam adicionados de uma só vez.

'git commit -m "nome do commit desejado" - esse comando criará o primeiro commit.

feito isso, pode ser que apareça um erro de autor desconhecido, se isso acontecer, deverá ser executado o seguinte comando:

'git config --global user.email "sofia.deniz@metam.com.br" ' - ou o e-mail desejado.

em seguida:

'git config --global user.name "Sofia Deniz" - para user

voltando para o comando:

'git commit -m "primeiro commit"

'git branch -M main'

'git remote add origin https://github.com/sofia.deniz@metam.com.br/numero-secreto.git' - comando que gerará o link que fará a conexão do git com o console do VS Code, versão HTTP, mas a recomendada é SSH, então:

'git remote add origin git@github.com:sofia.deniz@metam.com.br/numero-secreto.git'

'git push -u origin main' - serve para "push" enviar para o git.

um erro comum após a execução dos comandos acima, é o git não reconhecer a máquina utilizada para fazer a conexão entre ambos, pra isso é necessário realizar os ajustes no próprio git.

clicando no canto da tela do lado direito, vamos no nosso perfil, depois no menu a esquerda, clicamos em 'SSH and GPG keys', em seguida clicar no botão "New SSH key", depois ele pede um título que podemos escolher, recomendação "notebook pessoal", depois tem as opções de chave, é a de autenticação, e em seguida é pedido uma chave da qual podemos pegar no terminal ou na nossa próproia máquina. que é entrando na pasta do usuário e procurar uma sala oculta chamada ".ssh", se a pasta estiver vazia, é necessário realizar pelo terminal, usando o comando:

'ssh-keygen -t ed25519 -C "sofia.deniz@'metam.com.br" - depois só ir dando enter até a chave ser gerada, agora podemos pegar na pasta, irá ter 2 arquivos, uma chave pública e outra privada, para pegar, é necessário clicar no arquivo e abrir, normalmente com o bloco de notas, utilizaremos a pública, depois podemos utilizar o outro comando: 'git push -u origin main' depois podemos ir para a página do repositório ver se deu certo! :) se tiver aparecido os arquivos, deu certo <3

Clonando repositório:

no repositório, quando clicamos no botão "code", aparece a opção de zipar, abrir com github desktop e clone, se só fizermos o download do zip, vamos perder os comandos já executamos no terminal do vs code, então precisamos clonar. Para isso, será necessário copiar o linbk que o github dá, depois abrirmos a biblioteca > área de trabalho > botão direito do mouse + abrir o terminal do windows/linux nesse caso, no terminal iremos executar:

'git clone (+ o link coipiado)

Depois será poissível ver uma pasta na sua máquina com os arquivos clonados.

Commit (Remember :')

comandos:

git status - para mostrar qual o status atualizado do repositório atual, últimas alteraçõeas, etc

quando é realizado um'a alteração, quando executarmso git status, as últimas alterações aparecerão em vermelho, pois não foram adicionadas ainda, para isso é necessário adicionar o comando:

git add . depois podemnos dar git status - aparecerá em verde as últimas alterações.

after that, damos o commit:

git commit -m "Tal mudança" - m significa mensagem, no repositório nomear os commits ajudarão a saber quais mudanças foram feitas.

git log - mostra o histórico dos commits que foram feitos, mostra o autor, data e a mensagem do commit.

git remote - mostra quais repositórios temos

git push origin main

git pull origin main - serve para trazer os demais commits para o meu repositório local

git log - para confirmar se o commit veio para o repositório local

Existe uma forma de fazer um "commit" sem utilizar comandos, utilizando o terceiro icon do menu lado direito do vs code. clicando no mesmo, é possível criar um commit e sincronizá-lo apenas com um botão.

caso eu queira fazer um ctrl z do último commit, fazer com que ele volte para o que era antes, utilizarei:

'git log' - mostra o histórioco juntamente com o hash, copiar o hash do commit desejado.

'git revert rfu83htyurfhui23qj'

o histórico vai aparecer que foi resetado.

para apagar:

fazer um commit

git commit -m "remove título" ' para fazer o commit

git reset --hard [+i'd do commit anterior, que seria po id do commit que a gente quer voltar]

para alterar o nome do commit (caso tenha digitado errado opu só queira editar:

git commit --amend -m "Atualização"

README - basicamente é um arquivo opcional do qual você pode escrever e explicar como quiser.

se eu quiser fazer um rascunho, ou seja, ter um arquivo que não precisa ser lido, utilizamos 'git ignore', dependendo da tecnologia utilizada, podemos buscar por 'gitignore.io' no navegador, ele dará o código certo para linguagem especifica.

para também usar de rascunho, ou se quiser compartilhar alguma ideia isolada sua, podemos utilizar o gist.
